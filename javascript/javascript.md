1. 자바스크립트 언어가 다른 언어와 다른 특징을 3가지 서술하세요
- 동적 타입(dynamic type) : 변수 타입이 고정이 아니라, 실행 중에 바뀔 수 있습니다
- 단일 스레드 + 이벤트 루프 : 한 번에 하나의 작업만 처리하지만, 이벤트 루프와 큐로 비동기를 처리합니다
- 프로토타입 기반 객체지향 : 클래스 대신(ES6 전) 프로토타입 체인으로 상속을 구현할 수 있습니다

2. 변수에 대해 설명하고 let, const의 차이점을 서술하세요
- 변수: 값을 저장하기 위한 이름이 있는 메모리 공간
- let : 블록 스코프, 재할당 가능
- const : 블록 스코프, 재할당 불가 (단, 객체/배열 내부 값 수정은 가능)

3. 호이스팅 개념에 대해 설명해주세요
- 변수/함수 선언이 스코프의 최상단으로 끌어올려진 것처럼 동작하는 현상
- var, 함수 선언문, let, const 모두 호이스팅 대상이지만 var는 undefined로 초기화됨
- let, const는 초기화 전까지 일시적 사각지대(TDZ) 에 있어 접근 시 에러

4. 클로저 개념에 대해 설명해주세요
- 함수가 선언될 때의 외부 변수 환경을 기억해서, 나중에 호출되더라도 그 변수에 접근할 수 있는 기능
- “내부 함수 + 그 함수가 선언될 당시의 외부 변수 묶음”이라고 이해하면 쉬움

5.  '==', '==='의 차이점에 대해 설명해주세요
- == (느슨한 비교) : 타입이 다르면 형 변환 후 값만 비교
- === (엄격한 비교) : 타입과 값 모두 같아야 true

6. 자바스크립트의 비동기 처리 방법에 대해 말씀해주세요
- 콜백 함수(callback) : 함수에 다른 함수를 인수로 전달해서 비동기 완료 후 호출
- Promise : then, catch로 비동기 결과를 다루는 객체
- async / await : Promise 기반 비동기를 동기 코드처럼 읽기 쉽게 작성

7. 프로토타입에 대해 설명해주세요
- 모든 객체는 부모 역할을 하는 프로토타입 객체를 가짐
- 객체에서 프로퍼티를 찾을 때 없으면 프로토타입 체인을 따라 위로 올라가며 찾는다
- 이 연결 구조를 프로토타입 체인이라고 하고 최상단은 Object.prototype 이다

8.  this 키워드의 작동 방식과 this의 적용 범위에 대해 말씀해 주세요
- this는 함수를 어떻게 호출했는지에 따라 값이 달라짐
- 전역(브라우저) : window (strict 모드에서는 undefined)
- 객체 메서드 호출 : 그 객체를 가리킴
- 생성자 함수(new) : 새로 생성된 인스턴스
- call / apply / bind : 명시적으로 지정한 객체
- 화살표 함수 : 자신을 감싸는 외부 스코프의 this를 그대로 사용 (lexical this)

9. 스코프에 대해 설명하세요
- 변수를 어디서까지 접근할 수 있는지 범위
- 전역 스코프: 어디서나 접근 가능
- 함수 스코프: 함수 내부만
- 블록 스코프: {} 내부 (let, const)
- js는 렉시컬 스코프를 사용: 함수를 "어디서 정의했는지”에 따라 상위 스코프가 결정됨

10. 콜백지옥에 빠지지 않기 위해 어떻게 해야 하는지 설명해주세요
- Promise 사용 : then 체이닝으로 들여쓰기 줄이기
- async / await 사용 : 동기 코드처럼 위에서 아래로 읽히게 작성
- 함수 분리 : 긴 콜백을 여러 개의 작은 함수로 분리

11. var 변수 타입을 지양해야 하는 이유를 말씀해주세요
- 함수 스코프 : 블록 스코프가 아니어서 예측하기 어려운 버그 가능
- 호이스팅 + 즉시 초기화 : 선언 전에 접근해도 undefined로 나와서 실수 숨김
- 중복 선언 허용 : 같은 이름을 여러 번 선언 가능 > 유지보수 어려움

12. 동기와 비동기에 대해 말씀해주세요
- 동기(Synchronous)
하나의 작업이 끝날 때까지 다음 작업을 기다림
코드가 위에서 아래로 순서대로 실행

- 비동기(Asynchronous)
작업을 맡겨두고 완료되면 나중에 콜백/Promise/async-await으로 결과 처리
오래 걸리는 작업(API 호출, 타이머 등) 중에도 다른 코드 실행 가능

13. 이벤트 루프에 대해 설명해주세요
- JS는 단일 스레드이지만 이벤트 루프 덕분에 비동기를 처리함
ㅇ 흐름
- 호출 스택(Call Stack)에서 동기 코드 실행
- 비동기 작업은 브라우저/Node API에 맡김
- 작업이 끝나면 콜백을 태스크 큐/마이크로태스크 큐에 넣음
- 이벤트 루프가 스택이 비었는지 확인하고 비었으면 큐에서 콜백을 가져와 실행

마이크로태스크 큐(Promise 등)가 일반 태스크 큐(setTimeout 등)보다 우선순위가 높다.
